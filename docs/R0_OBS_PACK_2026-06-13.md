**Step 1 — Read `volatility_baseline_cache.json` in route:**
In `paper-performance/route.ts`, inside the try block after `const latest = await readLatest()`:
```javascript
// OBS-02: vol baseline sample diagnostic (read-only)
let volBaselineDiagnostic = null;
try {
  const { dir } = await import("@/lib/readLatest").then(m => m.resolveRuntimeDir());
  const path = await import("path");
  const fs = await import("fs/promises");
  const volCachePath = path.join(dir, "volatility_baseline_cache.json");
  const raw = await fs.readFile(volCachePath, "utf-8").catch(() => null);
  if (raw) {
    const cache = JSON.parse(raw);
    const samples = cache?.samples_1h ?? null;
    const required = cache?.required_points?.for_baseline_50 ?? 50;
    volBaselineDiagnostic = {
      samples1h: samples,
      required: required,
      volState: cache?.vol_state ?? null,
      volStateReliable: typeof samples === "number" && samples >= required,
      baselineReadiness: typeof samples !== "number" ? "UNKNOWN"
        : samples >= required ? "READY"
        : samples >= 24 ? "BUILDING"
        : "INSUFFICIENT",
      atrRatio: cache?.atr_ratio ?? null,
      bbwRatio: cache?.bbw_ratio ?? null,
      warningUnreliable: typeof samples === "number" && samples < required,
    };
  }
} catch { /* vol baseline not available — safe to skip */ }
// Attach to paperLoopDiagnostics
if (paperLoopDiagnostics) {
  (paperLoopDiagnostics as any).volBaselineDiagnostic = volBaselineDiagnostic;
}
```

### Phase OBS-03 — Hysteresis Readiness Diagnostic (Design-Only)

**Finding:** The canonical regime engine re-computes fresh each time from raw candles.  
There is NO flip history stored anywhere — no log file, no jsonl, no state field.

**Why it cannot be implemented now:**  
To detect flip-flop (V-04 from R-0 audit), we would need to:
1. Store the last N canonical regime results with timestamps
2. Count flips within a rolling window (e.g., 1h)
3. This requires writing to a new state file (e.g., `regime_history_cache.json`)

**Writing a new state file is NOT allowed in this OBS pack** (rule: do NOT modify runtime JSON/JSONL).

**Recommended next step (future approved phase):**
- Add `regime_history_cache.json` with rolling 1h regime snapshots (max 12 entries × 5min)
- Writer: server-side, updated every 5 min alongside `market_snapshot.json`
- Reader: dashboard `paperLoopDiagnostics` reads it and emits `regimeFlipCount1h` + `REGIME_UNSTABLE: bool`
- UI: badge on CanonicalMarketRegimeCard shows "⚠️ Regime unstable (N flips/1h)"

**Required governance approval before implementing:**  
Approve writing `regime_history_cache.json` as a new runtime output file.

**Status: DESIGN_ONLY — no code change in this phase.**
