# Runtime Files Git Policy

## Purpose
Prevent Git pulls on Plesk from overwriting fresh runtime JSON, JSONL, and TXT files generated by the backend and snapshot process.

## Runtime Root
`BINGX_AGENT_DIR=<PROJECT_ROOT>`

Examples:
- server: `/var/www/vhosts/ob-gate.com/httpdocs`
- local: `C:\2025\web-69\ob-gate17-200369\httpdocs`

- `latest_decision.json`
- `latest_decision_agent.json`
- `JSON_Agent.json`
- `latest_step2.txt`
- `klines.json`
- `orderbook_snapshot.json`
- `open_interest_snapshot.json`
- `oi_history_cache.json`
- `news_context.json`

- `volatility_baseline_cache.json`
- `funding_snapshot.json`
- `plan_status.json`
- `plan_status_state.json`

- `plan_history.jsonl`
- `scheduler_heartbeat.json`
- `alert_summary.json`

- `dashboard/public/data/*.json`
- `dashboard/public/data/*.jsonl`

## Why
- These files are generated by backend/runtime processes.
- Plesk may contain fresher runtime state than Git.
- `git pull` must not overwrite live market state, plan state, or paper evidence.
- Examples must use `.example.json` or `.example.jsonl` names only.

## Correct Workflow
1. Pull code from Git.
2. Do not overwrite runtime JSON, JSONL, or TXT files.
3. Let backend/snapshot regenerate runtime files.
4. Dashboard/API readers read runtime files from `<PROJECT_ROOT>`.
5. Commit only code, config, docs, and example fixtures.

## Emergency Recovery
If a runtime file was overwritten:
1. Stop the app or snapshot process if needed.
2. Restore from backup if available.
3. Rerun snapshot.
4. Verify `/api/health`, `/api/plan-status`, and `/api/runtime-audit`.

## Git Cleanup Rule
If a runtime file was previously tracked, remove it from Git index only:

```bash
git rm --cached <runtime-file>
```

Never delete the server runtime file during Git cleanup.
